App Description Document

Name: ClimaMatters.com

Purpose

The app's purpose is to provide weather information based on the user's location. The weather data is provided through API by the OpenWeatherMap services, owned by OpenWeather Ltd, which provides global weather data for any location from different sources.

Features

Current Weather

Displays the current temperature and weather description based on the user's latitude and longitude provided by geolocation data from the browser used. If the user clicks on the "+" icon, the app shows the following additional information about the weather:

  1. Humidity
  2. Wind speed
  3. Wind Chill (only if the temperature is less than 50 and the wind speed is greater than 3)

Daily Forecast for 7 Days

National Weather Alerts (only if alerts are active when accessing the app)

How to Use the Weather Application

On first access, for privacy reasons, the user's operating system will prompt the user to allow location access. If the user accepts, the browser will use the functionality available on the device to access location information using the Geolocation API and will display weather information based on location data. If the user's browser does not support the functionality, the following message will be displayed: "Geolocation is not supported by this browser". If the first access, the app will show the sentence "it seems like your first time here. Welcome!" at the bottom of the page. On subsequent accesses, the app will show the user's last date and last location when using the app.

CSS Features

Transitions and Transforms

  1. Page background.

    2. When loading the web site, a CSS animation is used to change the background color of the page.

  2. Web site icon

    3. When moving the mouse over the icon at the top of the website, the CSS transition feature is used to change the size of the website icon.

  3. Fieldset borders

    4. When moving the mouse inside an HTML element fieldset, the CSS Hover selector is used to round the edges and perform a transition of the color used in the image's background.

Events

When moving the mouse inside a fieldset HTML element, the onmouserover event executes a javascript function that modifies the background color of images inside the fieldset by CSS.

Design process

The design process was designed to simplify the visualization of weather information. Initially, the web application planning was developed, including the initial choices for color scheme, typography and wireframes. Then, when writing the code, tweaks were made to the design to simplify the app creation process. Finally, minor adjustments were made to the colors used and font sizes to improve the user experience.

Application of coding topics from the course

Document Object Model

The Document Object Model (DOM) was used in the project to create, read and change properties of HTML elements.

Javascript Functions

The Javascript functions were used to perform particular tasks and execute actions based on events.

JSON

The JSON format was used to return the data used in the application using the fetch methods.

CSS

CSS features were used to transform and transition HTML objects in response to mouse click or page load events.

LocalStorage

The LocalStorage was used to store and modify information in the user's browser such as the date and location of the last visit to the website.
